screen.java

screen.java is a class that will handle the screen of your game (essentially, it IS the screen of your game). As well as creating your game's screen, this class also has some useful methods that will help with visuals and memory control.

clear();

The clear(); method is pretty simple; all it does is clear the screen. This makes sure that the game doesn't overload and crash from rendering too many things at once.

Usage:

clear();

drawRect();

The drawRect(); method draws the border of a rectangle on your screen. This is usually used to draw out visible hitboxes or areas. This method takes the parameters:

  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • int width - # of pixels wide (x-axis)
  • int height - # of pixels tall (y-axis)
  • int color - color of the outline (in hexidecimal format)

Usage:

drawRect(0, 0, 10, 20, 0x000000);
// makes the border of a rectangle that starts at (0, 0) which is 10 px wide and 20 px tall and is black

fillRect();

The fillRect(); method functions exactly like the drawRect(); method, except instead of making the outline of a rectangle, it will make a filled in rectangle. This is usually used to make backgrounds or show areas. This method takes the parameters:

  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • int width - # of pixels wide (x-axis)
  • int height - # of pixels tall (y-axis)
  • int color - color of the outline (in hexidecimal format)
fillRect(0, 0, 10, 20, 0x000000);
//makes a solid rectangle that starts at (0, 0) which is 10 px wide and 20 px tall and is black

fillRectBlend();

The fillRectBlend(); method functions exactly the same as the fillRect(); method, except that it makes the rectangle slightly transparent. This method takes the parameters:

  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • int width - # of pixels wide (x-axis)
  • int height - # of pixels tall (y-axis)
  • int color - color of the outline (in hexidecimal format)

Usage:

fillRect(0, 0, 10, 20, 0x000000);
// makes a solid rectangle that starts at (0, 0) which is 10 px wide and 20 px tall and is black, but transparent

drawString();

There are several possible arguments for this method -- each one taking different parameters. The first use of the drawString(); method is pretty self explanatory -- it will draw a string. This method takes the parameters:

Args list 1:

  • String string - whatever string you want
  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
drawString("Hi", 0, 0);
//makes a string that says "Hi" that will start at (0, 0)

Args list 2:

  • String string - whatever string you want
  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • Font font - the font you want the text to be in
  • Color color - the color you want the text to be
drawString("Hi", 0, 0, "Rockwell", Color.black);
//makes a string that says "Hi" in black using the Rockwell font and will start at (0, 0)

Args list 3:

  • String string - whatever string you want
  • Rectangle rect - a rectangle that the string will center itself in
drawString("Hi", r);
//with r being a rectangle defined somewhere else in your code, "Hi" will be centered in it

drawTexture();

The drawTexture(); method is also quite self-explanatory. It will render a texture! This method takes the parameters:

Args list 1:

  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • Texture texture - the texture (image) you want to load
drawTexture(0, 0, "/sprites/image.png");
//draws the image starting at the co-ordinates (0,0)

Args list 2:

  • int x - x co-ordinate of start
  • int y - y co-ordinate of start
  • Texture texture - the texture (image) you want to load
  • boolean flipX - if true, will flip the texture along the x-axis
drawTexture(0, 0, "/sprites/image.png", false);
//draws the image starting at the co-ordinates (0,0) and does not flip it

window.java

window.java is a class that creates and handles the window of your game.

setWindowSize();

The setWindowSize(); method simply sets the size of your game's window. This method takes the parameters:

  • int width - how wide you want your window to be -- usually a multiple of 32 or 64
  • int height - how tall you want your window to be -- usually a multiple of 32 or 64
setWindowSize(960, 540);
//your game window will be 960x540 px
